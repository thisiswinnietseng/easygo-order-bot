⚙️ 開始前準備
🪪
你的 be2 帳號 / 密碼
每個人用自己的 be2 帳密登入，不共用同一組。勾選「記住帳號」後重整頁面會自動帶入帳號（密碼不會被記住，每次都要重新輸入）。
🔑
EasyGo 供應商帳號
EasyGo（供應商平台）用的是公司共用帳號，已經設定在程式的 .env 檔案裡，不用在網頁上填，也不需要每個人申請一組。
📋 怎麼使用
  1. 填好你的 be2 帳號密碼
  2. 把要訂購的 KKday 訂單編號貼進文字框，一行一筆，可以一次貼很多筆（批次處理）
  3. 按「🚀 開始執行」，會跳出 Chrome 視窗讓你全程看著跑，不用手動操作
  4. 每一筆都會顯示結果卡片：成功 中止 失敗，點開可以看詳細執行過程
  5. 全部跑完後可以按「📊 下載 Excel」，匯出這批的執行紀錄
💡 批次處理時，單筆訂單失敗或中止不會影響其他筆，會繼續往下跑完剩下的訂單。
🔄 執行邏輯與步驟
1️⃣
be2 端：抓資料 + 領單
登入 be2 → 開啟訂單 → 讀取導覽語系、出發日期、訂單總成本、旅客人數、旅客姓名、訂購人資料、旅客備註 → 點擊「OP領取鍵」→「已訂出待供應商回覆」。
2️⃣
EasyGo 端：下單 + 付款
登入 EasyGo → 用固定商品ID開商品確認頁 → 選日期 = be2 出發日期 → 類別固定選「30%」、依人數填數量 → 系統算出的結算金額須與 be2 訂單總成本一致才會繼續 → 填確認信箱（固定 op-ib@kkday.com）、簡体中文、游客姓名/手機/國籍/緊急聯絡（固定 X）→ 內部備註填入 KKday 訂單編號 → 提交訂單 → 立即支付。
3️⃣
be2 端：回填憑證
回 be2「發送Voucher」分頁 → 填入 EasyGo 訂單ID → 產生 KKday 憑證 → 寄送 Voucher 給旅客 → 標記「已寄出Voucher」→ 這筆訂單完成！
⚠️ 注意事項
不要對同一張已經下單付款過的訂單重複執行。EasyGo 下單目前沒有防重複下單機制，同一張 KKday 訂單如果已經在 EasyGo 建立過訂單，再跑一次會產生第二筆重複訂單（重複扣款）。
「導覽語系」非 zh-tw、「旅客備註」有內容，都不會中止流程，還是會正常送出訂單，只會把這兩項實際值記錄在執行紀錄跟 Excel 匯出裡，供你事後回顧（旅客備註內容不會被帶入 EasyGo 訂單，因為 EasyGo 表單本身沒有對應欄位）。
金額比對只在「填人數」那一步做一次，跟 be2「訂單總成本」不一致就會中止該筆並記錄原因；付款頁不再重複比對。
