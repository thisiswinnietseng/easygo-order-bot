🚀 JPH 訂購自動化 使用說明書

be2 抓單 → JPH 下單付款 → 回填憑證 · 操作指南

⚙️ 開始前準備
🪪
你的 be2 帳號 / 密碼
每個人用自己的 be2 帳密，填在自己電腦的 .env 檔案裡（BE2_USERNAME / BE2_PASSWORD），網頁上不用輸入。不要把自己的帳密給別人用，每個人應該用自己的 .env 填自己的帳號，這樣 be2 上的操作紀錄才會顯示是本人執行的。
🔑
JPH 供應商帳號
JPH（供應商平台）用的是公司共用帳號，一樣填在 .env 裡（EASYGO_PASSWORD，跟主管或 Winnie 索取密碼）。
📝
怎麼編輯 .env
.env 是隱藏檔案，在自己電腦裡打開終端機（Terminal），貼上：
open -e ~/Desktop/easygo-order-bot/.env
會用「文字編輯」打開，把 BE2_USERNAME、BE2_PASSWORD、EASYGO_PASSWORD 後面都填好，存檔即可。.env 只留在自己的電腦上，不會上傳、也不會被別人看到，所以填自己的帳密是安全的。
📋 怎麼使用
  1. 第一次使用要先照上面說明把 .env 填好（見「開始前準備」）
  2. 把要訂購的 KKday 訂單編號貼進文字框，一行一筆，可以一次貼很多筆（批次處理）
  3. 按「🚀 開始執行」，會跳出 Chrome 視窗讓你全程看著跑，不用手動操作
  4. 每一筆都會顯示結果卡片：成功 中止 失敗，點開可以看詳細執行過程
  5. 全部跑完後可以按「📊 下載本次結果」，匯出這批的執行紀錄
💡 批次處理時，單筆訂單失敗或中止不會影響其他筆，會繼續往下跑完剩下的訂單。
📜「下載歷史紀錄」區塊會匯出伺服器上累積的所有執行紀錄（不只這次畫面上看到的），就算重新整理頁面也不會遺失；可以選填開始/結束日期只匯出某段期間，兩個都不填就是全部。
🔄 執行邏輯與步驟
1️⃣
be2 端：抓資料 + 領單
登入 be2 → 開啟訂單 → 讀取導覽語系、出發日期、訂單總成本、旅客人數、旅客姓名、訂購人資料、旅客備註 → 點擊「OP領取鍵」→「已訂出待供應商回覆」。
2️⃣
JPH 端：下單 + 付款
登入 JPH → 用固定商品ID開商品確認頁 → 選日期 = be2 出發日期 → 類別固定選「30%」、依人數填數量 → 系統算出的結算金額須與 be2 訂單總成本一致才會繼續 → 填確認信箱（固定 op-ib@kkday.com）、簡体中文、游客姓名/手機/國籍/緊急聯絡（固定 X）→ 內部備註填入 KKday 訂單編號 → 提交訂單 → 立即支付。
3️⃣
be2 端：回填憑證
回 be2「發送Voucher」分頁 → 填入 JPH 訂單ID → 產生 KKday 憑證 → 寄送 Voucher 給旅客 → 標記「已寄出Voucher」→ 這筆訂單完成！
⚠️ 注意事項
不要對同一張已經下單付款過的訂單重複執行。JPH 下單目前沒有防重複下單機制，同一張 KKday 訂單如果已經在 JPH 建立過訂單，再跑一次會產生第二筆重複訂單（重複扣款）。
「導覽語系」非 zh-tw、「旅客備註」有內容，都不會中止流程，還是會正常送出訂單，只會把這兩項實際值記錄在執行紀錄跟 Excel 匯出裡，供你事後回顧（旅客備註內容不會被帶入 JPH 訂單，因為 JPH 表單本身沒有對應欄位）。
金額比對只在「填人數」那一步做一次，跟 be2「訂單總成本」不一致就會中止該筆並記錄原因；付款頁不再重複比對。
